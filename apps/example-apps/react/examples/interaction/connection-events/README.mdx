React Flow 在连接过程中会触发不同事件，您可利用这些事件以多种方式更新界面或流程。下方示例展示了各事件的触发时机。

当通过从手柄拖拽创建**新连接**时，将按顺序触发以下事件：

- [`onConnectStart`](/api-reference/react-flow#on-connect-start) 触发时携带鼠标事件对象及包含源节点（可能包含源手柄ID）和手柄类型的参数对象

- [`onConnect`](/api-reference/react-flow#on-connect) 仅当连接释放至[可连接手柄](/api-reference/components/handle#is-connectable)时触发。参数为完整的[连接对象](/api-reference/types/connection)，包含源/目标节点及手柄ID（如存在）

- [`onConnectEnd`](/api-reference/react-flow#on-connect-end) 在连接释放时触发（无论成功与否），参数为鼠标事件对象

当通过拖拽现有边进行**重连接**时，将按顺序触发以下事件：

- [`onReconnectStart`](/api-reference/react-flow#on-reconnect-start) 在拾取[可重连边](/api-reference/types/edge#reconnectable)时触发。调用时会传入鼠标事件、正在重连的边对象以及稳定手柄的类型。

- [`onConnectStart`](/api-reference/react-flow#on-connect-start) 触发机制同上。

- [`onReconnect`](/api-reference/react-flow#on-reconnect) 当边被释放到[可重连](/api-reference/types/edge#reconnectable)的手柄上时触发。调用时会传入旧的[边对象](/api-reference/types/edge)和新的[连接对象](/api-reference/types/connection)。

- [`onConnectEnd`](/api-reference/react-flow#on-connect-end) 触发机制同上。

- [`onReconnectEnd`](/api-reference/react-flow#on-reconnect-end) 在边被释放时触发（无论重连是否成功）。调用时会传入鼠标事件、被拾取的边对象以及稳定手柄的类型。
