# 基础节点

一个带有基础样式的节点包装器，用于为应用程序中的所有节点创建统一的设计风格。
类似于 [shadcn ui 的卡片组件](https://ui.shadcn.com/docs/components/card)，该组件文件导出以下内容：

- `BaseNode` 主容器
- `BaseNodeHeader` 容器（通常用于添加操作和 `BaseNodeHeaderTitle`）
- `BaseNodeContent` 容器（用于添加节点的主要内容）
- `BaseNodeFooter` 容器（可用于添加额外信息或可见操作）

如需微调自定义组件与拖拽、滚动等交互行为，
React Flow 提供了 [多个 CSS 实用类](/learn/customization/utility-classes)

对于节点中的交互式组件（如按钮），应使用 React Flow 的 `nodrag` [实用类](/learn/customization/utility-classes)，
以防止用户在操作按钮或滑块时意外拖动流程中的节点。

## 主题定制

要自定义节点的视觉外观，可直接使用
[Tailwind CSS](https://tailwindcss.com/) 类。所有 React Flow
组件都基于 [shadcn UI](https://ui.shadcn.com/)，建议遵循
[shadcn UI 主题指南](https://ui.shadcn.com/docs/theming) 来定制应用程序中的排版和颜色等样式。

不过在大多数情况下，开发自定义节点时，你可能只需要添加自定义的 Tailwind CSS 类。所有 `BaseNode` 组件都只是 `<div>` 元素的轻量级封装。

例如，要根据假设的执行状态更改节点的边框颜色，你可以传递额外的 `className`：

```tsx
// Assuming your component is receiving a `data` prop
export const BaseNodeSimpleDemo = memo(({ data }: NodeProps) => {
  return (
    <BaseNode
      className={cn('w-[350px] p-0 hover:ring-orange-500', {
        'border-orange-500': data.status === 'loading',
        'border-red-500': data.status === 'error',
      })}
    >
      {/* Your custom node definiton goes here */}
    </BaseNode>
  );
});
```
