# 概述

React Flow 的核心功能是创建交互式流程图——由边线连接的节点集合。

### 节点

React Flow 默认提供几种节点类型，但真正的魔力在于自定义功能。
您可以完全按照需求设计节点——无论是嵌入交互式表单元素、
展示动态数据可视化，还是配置多个连接手柄。React Flow 提供基础框架，
剩下的就交给您的想象力发挥。

我们提供了创建[自定义节点](/learn/customization/custom-nodes)的指南，
您也可以在[节点](/api-reference/types/node)参考文档中找到所有自定义选项。

### 连接手柄

连接手柄（其他库中常称为"端口"）是边线与节点的连接锚点。
默认情况下，它们显示为节点顶部、底部、左侧或右侧的灰色圆形。
但它们本质上是 `div` 元素，可以任意定位和样式化。
创建自定义节点时，您可以根据需要添加任意数量的手柄。
更多信息请参阅[手柄](/learn/customization/handles)页面。

### 边线

边线用于连接节点。每条边线都需要有目标节点和源节点。
React Flow 内置了四种[边线类型](/examples/edges/edge-types)：
`default`（贝塞尔曲线）、`smoothstep`、`step` 和 `straight`。
边线本质上是可通过 CSS 样式化且完全可定制的 SVG 路径。若使用多个连接点，可分别引用它们来为节点创建多重连接。

与自定义节点类似，边线同样支持定制。常见的自定义边线应用场景包括：

- 添加删除边线的按钮
- 自定义路由行为
- 无法通过单一 SVG 路径实现的复杂样式或交互

更多详细信息请参阅[边线](/learn/customize/edges)页面。

### 连接线

React Flow 内置了通过点击拖拽连接点来创建新边线的功能。拖拽过程中显示的临时边线称为连接线。连接线同样具备与边线相同的四种内置类型，且支持自定义配置。相关属性配置请查阅[连接属性](/api-reference/react-flow#connection-line-props)文档。

### 视口

React Flow 的所有内容都包含在视口(viewport)内。每个节点都具有 x 和 y 坐标，用于表示其在视口中的位置。视口本身包含 x、y 和缩放(zoom)三个参数值。当您拖动面板时，实际上是在改变 x 和 y 坐标；而进行缩放操作时，则是在调整缩放级别。
