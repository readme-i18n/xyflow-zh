# 实用工具类

React Flow 提供了多个内置的实用 CSS 类，可帮助您微调自定义元素内的交互方式。

## `nodrag`

为元素添加 `nodrag` 类可确保与其交互时不会触发拖拽操作。这对于按钮或输入框等不应在点击时启动拖拽的元素特别有用。

节点默认带有 `drag` 类名。但该类名可能会影响自定义节点内事件监听器的行为。为避免意外行为，请为带有事件监听器的元素添加 `nodrag` 类名。这样既能阻止默认拖拽行为，也能防止点击带此类的元素时触发默认节点选择行为。

```tsx
export default function CustomNode(props: NodeProps) {
  return (
    <div>
      <input className="nodrag" type="range" min={0} max={100} />
    </div>
  );
}
```

## `nopan`

若画布中的元素未阻止鼠标事件传播，点击并拖动该元素会导致视口平移。添加 "nopan" 类可阻止此行为，该属性允许您修改该类的名称。

```tsx
export default function CustomNode(props: NodeProps) {
  return (
    <div className="nopan">
      <p>fixed content...</p>
    </div>
  );
}
```

## `nowheel`

如果自定义元素包含可滚动内容，您可以应用 `nowheel` 类。这会在自定义节点内滚动时禁用画布的默认平移行为，确保只有内容滚动而不会移动整个画布。

```tsx
export default function CustomNode(props: NodeProps) {
  return (
    <div className="nowheel" style={{ overflow: 'auto' }}>
      <p>Scrollable content...</p>
    </div>
  );
}
```

应用这些实用类可以帮助您精细控制交互行为。您可以通过 React Flow 的
[样式属性](/api-reference/react-flow/#style-props)
来自定义这些类名。

<Callout type="info">
  创建自定义节点时，您还需要记住为其添加样式！
  与内置节点不同，自定义节点没有默认样式，因此可以使用您偏好的任何
  样式方法，例如 [Tailwind CSS](/examples/styling/tailwind)。
</Callout>
